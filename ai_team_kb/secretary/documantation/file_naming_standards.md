# File Naming Standards

## Purpose
To ensure every file the assistant creates, saves, or manages follows a consistent, searchable, professional naming standard.

Clean file names prevent confusion, make retrieval instant, and support professional workflows.

These standards apply to **all**:
- Documents  
- PDFs  
- Images  
- Videos  
- Audio files  
- Deliverables  
- Notes  
- Exports  
- Attachments  

---

# 1. Core Naming Structure

All file names must follow this pattern:

[ProjectOrTopic][Description][YYYY-MM-DD]_vX.X.[ext]


Where:

**ProjectOrTopic**  
- Company, client, or project name  
- Example: “Erdie”, “Deadpilot”, “MarketingPlan”, “ClientA”  

**Description**  
- What the file actually is  
- Example: “Proposal”, “Invoice”, “MeetingNotes”, “ProductSheet”  

**YYYY-MM-DD**  
- Always ISO date format  
- Never use US-style mm-dd-yyyy  

**vX.X**  
- Version number  
- No “final_final_FINAL” garbage  

**ext**  
- pdf, docx, xlsx, png, mp4, etc.

---

# 2. Examples (Good)

MarketingPlan_Q1Strategy_2025-03-12_v1.1.pdf
Deadpilot_EP_PressRelease_2025-01-22_v2.0.docx
Erdie_TwistNPull_ProductSheet_2025-02-10_v1.0.pdf
ClientA_Contract_2025-03-01_v1.3.docx
MeetingNotes_TeamSync_2025-03-05_v1.0.md
Photoshoot_ReferenceImages_2025-02-27_v1.0.zip


---

# 3. Examples (Bad)

final.pdf
draft_v3_final_revised.pdf
update.docx
notes.png
IMG_2345.png
untitled.pdf


Anything unclear, generic, or messy = NOT allowed.

---

# 4. Versioning Rules

Never overwrite unless instructed by the user.

### Major versions:
- v1.0  
- v2.0  
Used when:
- Structure changes  
- Big content updates  
- Final drafts  

### Minor versions:
- v1.1  
- v1.2  
Used when:
- Small edits  
- Tweaks  
- Typos fixed  
- Formatting changes  

---

# 5. Capitalization Rules

Use **PascalCase** with underscores:

Correct:

ProjectProposal_NewBranding_2025-03-10_v1.0.pdf


Incorrect:

projectproposalnewbranding march10.pdf
Project proposal (final) v3.pdf


---

# 6. No Spaces, No Special Characters

Allowed:
- Letters  
- Numbers  
- Underscores  
- Dashes (for dates only)  

Forbidden:
- Spaces  
- &, %, $, #, @, !  
- Parentheses  
- Emojis  
- Mixed caps  

---

# 7. Image Naming Rules

Images used in projects must also follow structure:

ProjectName_ImageSubject_YYYY-MM-DD_v1.0.png
Erdie_TubeMockup_2025-02-01_v1.0.png
Deadpilot_PressPhoto1_2025-03-14_v1.0.jpg


Never upload images titled:
- IMG_0033
- Screenshot 2025-02-18 at 11.02.14 PM
- final edit.png

---

# 8. Meeting Notes Naming

Always:

MeetingNotes_[MeetingName]_[YYYY-MM-DD]_v1.0.md


Example:

MeetingNotes_InvestorCall_2025-03-20_v1.0.md


---

# 9. Task/Project Document Naming

Documents supporting major projects:

ProjectName_Deliverable_YYYY-MM-DD_v1.0


Example:

AlbumRelease_MarketingTimeline_2025-04-02_v1.0.xlsx


---

# 10. Success Criteria

File naming is “successful” when:
- You can find any file instantly  
- No conflicting names exist  
- No confusion between versions  
- External collaborators understand your files  
- File names look professional  
- Automatically sortable in any OS  

---

# Summary
A world-class assistant uses:
- Predictable naming  
- Clean structure  
- Version numbers  
- Datestamps  
- Descriptive labels  

This system makes every file easy to find, clear to understand, and impossible to lose.
