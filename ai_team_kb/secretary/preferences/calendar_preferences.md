# 4. Scheduling Rules

### **General Rules**
- Never schedule without confirming unless explicitly allowed  
- Always propose 2–3 options  
- Avoid scheduling over meals  
- Keep morning hours free unless user says otherwise  
- Avoid schedule overload  
- Protect user’s peak focus times  

### **Time Zone Handling**
- Always confirm the time zone  
- Convert automatically  
- Include time zone in all summaries  
- Warn user if conflicts exist  

### **Travel & Commute Time**
- Add commute buffers for in-person meetings  
- Prevent scheduling too close to travel windows  
- Add decompression time after returning  

---

# 5. Approval Rules

### Assistant Can Schedule Without Asking (if allowed)
- Routine check-ins  
- Internal team syncs  
- Recurring reminders  
- Non-critical administrative meetings  

### Assistant Must Ask Before Scheduling
- Anything with new clients  
- High-stakes calls  
- Interviews  
- Large group meetings  
- Meetings requiring preparation  
- Meetings outside normal working hours  

### Assistant Must NEVER Schedule:
- During user’s protected time  
- During known personal commitments  
- During rest periods  
- During travel windows  

---

# 6. Auto-Decline Rules

Assistant should automatically decline or propose alternatives when:
- Meeting is too early  
- Meeting interrupts deep work  
- Meeting violates energy patterns  
- Too many meetings already booked  
- Back-to-back overload would occur  

Assistant declines politely:
> “That time doesn’t work — here are some alternative options…”

---

# 7. Special Scheduling Scenarios

### **1. When two people request meetings at the same time**
Assistant should:
- Check priority  
- Offer alternatives  
- Communicate clearly  
- Avoid the user needing to get involved  

### **2. When the user forgets to mention a conflict**
Assistant checks calendar → warns immediately:
> “You already have something at that time.”

### **3. When travel is involved**
Assistant should:
- Add early departure buffer  
- Add return buffer  
- Protect day-before and day-after if long trip  

### **4. Under heavy workload**
Assistant reduces meetings automatically.

### **5. Sensitive or stressful meetings**
Assistant schedules:
- Later in the day  
- With more buffer time  
- When user’s energy is better  

---

# 8. Rescheduling Rules

### Assistant can reschedule without approval:
- Internal low-stakes meetings  
- Admin check-ins  
- Non-critical recurring meetings  

### Assistant must ask before rescheduling:
- Client meetings  
- Anything involving guests  
- High-importance calls  

### If the user requests rescheduling:
Assistant must:
1. Acknowledge  
2. Provide alternative times  
3. Handle notifications  
4. Ensure no conflicts  

---

# 9. Calendar Cleanliness Rules

Assistant ensures:
- No duplicated entries  
- All meetings have descriptions  
- All meetings have locations/links  
- All meetings have agendas  
- No event has vague titles like “Call”  
- Past events include notes (if required)  
- Unnecessary placeholders are removed  
- Long blocks are labeled clearly  

---

# 10. Success Criteria

Calendar management is “successful” when:
- User feels in control of their time  
- No meetings feel chaotic or back-to-back  
- Buffers are respected  
- Peak productivity is protected  
- Decisions are made automatically  
- Rescheduling is painless  
- Conflicts never occur  
- Deadlines sync with time availability  

---

# Summary
A world-class executive assistant manages the calendar like a second nervous system — protecting energy, respecting boundaries, optimizing flow, and ensuring the user stays focused and balanced.

This file ensures scheduling always aligns with the user’s natural working style.
